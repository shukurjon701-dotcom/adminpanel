# Деплой на Render — 100% бесплатно, без карты (бот + панель 24/7)

Render даёт бесплатный тариф без привязки карты. Единственный нюанс бесплатного
плана: сервис «засыпает» после 15 минут без запросов. Чтобы бот работал
круглосуточно, мы добавили эндпоинт `/health` и будем пинговать его бесплатным
сервисом **UptimeRobot** — так инстанс не засыпает.

> ⚠️ База данных (пользователи, история, «обучение» ИИ) на бесплатном Render
> **эфемерная**: она обнуляется при каждом передеплое/перезапуске. Это ты
> подтвердил как некритично. Если позже захочешь сохранность — переедем на
> бесплатную облачную Postgres (Neon/Supabase).

## Шаг 1. Залить код на GitHub (один раз)

Локальный git-репозиторий уже готов и закоммичен. Секреты (`.env`, база `*.db`)
в него НЕ попадают.

1. Заведи аккаунт на https://github.com (если ещё нет).
2. Создай **новый пустой репозиторий**: https://github.com/new
   - Имя: например `academy-arabic-bot`
   - Тип: **Private** (приватный) — можно.
   - **НЕ** добавляй README, .gitignore, лицензию (галочки не ставь).
   - Нажми *Create repository*.
3. Создай токен доступа (Personal Access Token):
   - Открой https://github.com/settings/tokens/new
   - Note: `render deploy`, Expiration: на твой выбор.
   - Отметь галочку **repo** (полный доступ к репозиториям).
   - *Generate token* → скопируй строку вида `ghp_...` (больше её не покажут).
4. Залей код. В строке Claude Code выполни (подставь свой логин, имя репо и токен):

   ```
   !cd "/Users/admin/Desktop/Новая папка 2" && git remote add origin https://<ТВОЙ_ЛОГИН>:<ТОКЕН>@github.com/<ТВОЙ_ЛОГИН>/academy-arabic-bot.git && git push -u origin main
   ```

   Если увидишь `Branch 'main' set up to track...` — код на GitHub. ✅

## Шаг 2. Развернуть на Render

1. Заведи аккаунт: https://dashboard.render.com/register
   (можно войти через тот же GitHub — карта НЕ нужна).
2. В дашборде: **New +** → **Blueprint**.
3. Подключи свой GitHub и выбери репозиторий `academy-arabic-bot`.
   Render сам найдёт файл `render.yaml`.
4. Render попросит ввести секреты (они помечены `sync: false`). Впиши значения
   из своего `.env`:
   - `TELEGRAM_BOT_TOKEN` — токен бота от BotFather
   - `GROQ_API_KEY` — ключ Groq
   - `ADMIN_USERNAME` — логин для входа в панель
   - `ADMIN_PASSWORD` — пароль для входа в панель
5. Нажми **Apply / Create**. Render соберёт и запустит сервис (2–4 минуты).
6. Когда статус станет **Live**, наверху будет адрес вида
   `https://academy-arabic-bot.onrender.com` — это твоя админ-панель
   (логин/пароль — те, что ввёл выше).
7. Проверь бота в Telegram: напиши ему `/start`.

---

## Шаг 3. Не давать сервису засыпать (UptimeRobot)

1. Заведи бесплатный аккаунт: https://uptimerobot.com (карта не нужна).
2. **+ New monitor**:
   - Monitor Type: **HTTP(s)**
   - Friendly Name: `academy bot`
   - URL: `https://<твой-адрес>.onrender.com/health`
   - Monitoring Interval: **5 minutes**
3. Save. Теперь UptimeRobot дёргает `/health` каждые 5 минут, инстанс не
   засыпает, и бот отвечает 24/7.

> Замечание: даже с пингом первый ответ после долгой паузы может задержаться
> на ~30 секунд (холодный старт). При активном пинге такое случается редко.

## Полезное

- **Логи**: в дашборде Render → твой сервис → вкладка **Logs**.
- **Сменить пароль панели**: Render → сервис → **Environment** → поменяй
  `ADMIN_PASSWORD` → сервис передеплоится.
- **Файлы `fly.toml` / `DEPLOY.md`** остались от старого варианта (Fly.io) —
  на Render не используются, можно игнорировать или удалить.
