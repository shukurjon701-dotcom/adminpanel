# Деплой на Fly.io (бот + админ-панель в одном контейнере, 24/7)

Всё уже подготовлено:
- `run_all.py` — запускает бота и веб-панель вместе, в одном процессе
- `Dockerfile` — как собрать образ
- `fly.toml` — конфиг Fly: постоянный диск `/data` для базы, машина не «засыпает»
- `.gitignore` / `.dockerignore` — секреты и локальная база не попадают в образ

## Что делаете вы (один раз)

1. Создайте аккаунт на https://fly.io/app/sign-up (нужна карта для проверки).
2. Войдите из терминала. В строке Claude Code наберите с восклицательным знаком:

   ```
   !~/.fly/bin/flyctl auth login
   ```

   Откроется браузер — подтвердите вход. После этого скажите мне «готово».

## Что дальше сделаю я (командами отсюда)

3. `flyctl launch --no-deploy` — создаст приложение и том (volume) для базы.
4. `flyctl secrets import < .env` — зальёт токены/пароли как секреты (значения не светятся в логах).
5. `flyctl deploy` — соберёт образ в облаке и запустит.
6. `flyctl status` / `flyctl logs` — проверю, что бот отвечает.

## После деплоя

- Бот работает 24/7 в Telegram.
- Админ-панель открывается по адресу вида `https://<имя-приложения>.fly.dev`
  (логин/пароль — из `.env`: `ADMIN_USERNAME` / `ADMIN_PASSWORD`).
- База (пользователи, история, «обучение» ИИ) хранится на постоянном диске и НЕ теряется при перезапусках.

## Полезные команды

```
~/.fly/bin/flyctl logs      # смотреть логи
~/.fly/bin/flyctl status    # состояние машины
~/.fly/bin/flyctl deploy    # передеплой после изменений в коде
~/.fly/bin/flyctl secrets set ADMIN_PASSWORD=новый_пароль   # сменить секрет
```
